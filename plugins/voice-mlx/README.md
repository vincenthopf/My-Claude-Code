# Voice MLX Plugin

Voice notifications and terminal status for Claude Code on macOS. Speaks summaries aloud using edge-tts (Microsoft neural voices), sets iTerm2 per-pane titles with LLM-generated summaries, and color-codes tabs by session status.

## Features

- **Voice notifications** — Speaks when Claude finishes, needs permission, or has a question
- **Tab color coding** — Tabs change color based on session status so you can see at a glance what each pane is doing
- **Terminal status titles** — Per-pane title bars show descriptive 6-12 word summaries of the current task
- **LLM-powered titles** — Inception Labs Mercury-2 API generates contextual titles (with local keyword fallback)
- **Banner alerts** — macOS notifications via terminal-notifier with click-to-focus
- **Audio locking** — Prevents overlapping speech from multiple sessions
- **Markdown stripping** — Strips `*`, `` ` ``, `_`, and `#` characters so TTS reads naturally

## Tab Colors

Status is conveyed through iTerm2 tab colors using escape sequences written directly to the TTY device. Each pane stores its own color independently.

| Status | Color | Hex | When |
|--------|-------|-----|------|
| WORKING | Blue | `#1a73e8` | User submits a prompt |
| DONE | Green | `#34a853` | Claude finishes responding |
| PERMISSION | Red | `#ea4335` | Claude needs permission to run a tool |
| QUESTION | Yellow | `#f9ab00` | Claude has a question for the user |
| ATTENTION | Orange | `#fa7b17` | Generic attention needed |

When multiple panes are in one tab, the tab bar shows the color of the focused pane. Colors persist until the next status change.

### How tab colors work

Standard escape sequences to stdout don't work inside Claude Code because it pipes stdout/stderr. Instead, the plugin:

1. Walks the process tree to find the session's TTY device (e.g., `/dev/ttys003`)
2. Writes the iTerm2 proprietary escape sequence directly to that device:
   ```
   \033]1337;SetColors=tab=RRGGBB\a
   ```
3. iTerm2 intercepts the sequence and applies the color to the correct pane

## Terminal Status Titles

Pane titles show descriptive summaries of what's happening in each session. Two sources:

### Mercury-2 API (recommended)

[Inception Labs Mercury-2](https://www.inceptionlabs.ai/) is a diffusion LLM that generates titles in ~0.6s. It produces titles like:

- `Adding voice plugin to My-Claude-Code GitHub repo`
- `Fixing auth token refresh in login API endpoint`
- `Researching iTerm2 tab color escape sequences`

Uses `reasoning_effort: "instant"` for minimal latency. Titles are 6-12 words, specific enough to tell you exactly what each pane is doing.

### Local fallback

Without an API key, titles are generated by extracting meaningful keywords from the prompt. Less descriptive but zero latency.

### How pane titles work

Like tab colors, pane titles can't be set via stdout. The plugin uses AppleScript via `osascript` to communicate with iTerm2 through Apple Events IPC:

```applescript
tell application "iTerm2"
    repeat with w in windows
        repeat with t in tabs of w
            repeat with s in sessions of t
                if tty of s is "/dev/ttys003" then
                    set name of s to "the title"
                end if
            end repeat
        end repeat
    end repeat
end tell
```

This targets the correct pane by matching the TTY device path, bypassing stdout entirely.

## Requirements

- macOS with iTerm2
- `edge-tts` — `pip install edge-tts` (or `uv add edge-tts`)
- `ffplay` — from ffmpeg (`brew install ffmpeg`)
- `terminal-notifier` — `brew install terminal-notifier` (optional, for banner alerts)
- Python 3.10+
- (Optional) `iterm2` Python package for click-to-focus notifications
- (Optional) Inception Labs API key for Mercury-2 powered titles

## Installation

Copy the plugin to your Claude Code plugins directory:

```bash
cp -r plugins/voice-mlx ~/.claude/plugins/custom/voice-mlx
```

Or symlink for easier updates:

```bash
ln -s "$(pwd)/plugins/voice-mlx" ~/.claude/plugins/custom/voice-mlx
```

### Terminal Status Setup

Prevent Claude Code from overriding pane titles:

```bash
echo 'export CLAUDE_CODE_DISABLE_TERMINAL_TITLE=1' >> ~/.zshrc
```

Enable per-pane title bars in iTerm2:

**Settings → Appearance → Panes → Show per-pane title bar with split panes**

### Mercury API (Optional)

For LLM-generated titles instead of keyword extraction:

```bash
echo 'your-api-key' > ~/.claude/inception_api_key
chmod 600 ~/.claude/inception_api_key
```

Get a key from [Inception Labs](https://www.inceptionlabs.ai/). The plugin uses the `mercury-2` model with `reasoning_effort: "instant"`.

Alternatively, set `INCEPTION_API_KEY` or `MERCURY_API_KEY` as an environment variable.

## Commands

- `/speak` — Enable voice feedback
- `/speak stop` — Disable voice feedback
- `/speak <voice>` — Change voice (e.g., `en-GB-RyanNeural`)
- `/speak voices` — List available voices
- `/speak prompt <text>` — Set custom voice instruction

## Configuration

Edit `~/.claude/voice.local.md`:

```yaml
---
voice: en-AU-WilliamNeural
enabled: true
notifications: true
prompt: "keep summaries casual"
---
```

| Field | Default | Description |
|-------|---------|-------------|
| `voice` | `en-AU-WilliamNeural` | Any edge-tts voice ID |
| `enabled` | `true` | Enable/disable voice feedback |
| `notifications` | `true` | Enable/disable macOS banner alerts |
| `prompt` | (none) | Custom instruction for voice style |

## Architecture

### Hook lifecycle

| Hook | Trigger | Terminal Status | Voice |
|------|---------|----------------|-------|
| UserPromptSubmit | User sends a prompt | Sets title + blue tab | Injects voice reminder into context |
| Stop | Claude finishes | Sets completion title + green tab | Speaks summary of response |
| Notification (permission) | Tool needs approval | Sets action title + red tab | Speaks what permission is needed |
| Notification (question) | Claude asks a question | Sets title + yellow tab | Speaks the question |
| Notification (idle) | Session waiting | No change (preserves previous) | Speaks "agent is idle" |

### Key technical details

- **TTY discovery**: Walks the process tree via `ps -o ppid=,tty=,comm=` to find the ancestor shell's TTY device. Cached per session in `~/.claude/terminal_status/`.
- **Audio locking**: File-based lock at `/tmp/voice-playback.lockdir` prevents overlapping speech. Includes stale lock detection and a 60-second watchdog.
- **Notification grouping**: Banner notifications are grouped by iTerm2 session UUID so new notifications replace old ones for the same pane.
- **State persistence**: Session state (TTY path, last summary) stored as JSON in `~/.claude/terminal_status/` with restricted permissions (0o700/0o600).
